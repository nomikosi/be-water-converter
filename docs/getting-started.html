General

This is a fun-made project. I always wanted to create an IntelliJ plugin. If you find any issues, feel free to create a PR at https://github.com/nomikosi/be-water-converter or message me directly.

If you have any other ideas about improvements, feel free to share those at nomikosi.consulting at gmail.com.

1. Install the plugin

Open your JetBrains IDE, go to Settings → Plugins → Marketplace, search for Be Water Converter, and click Install. Restart the IDE when prompted.

2. Open the tool window

Click the Be Water tab on the right side bar, or go to Tools → Be Water Converter.

3. Load your data

Paste text into the input editor, or load a file using one of these methods:

  • Drag and drop a file onto the input editor
  • Open a file from the toolbar button

The source format is auto-detected from the file extension.

4. Convert

Select the source format in the From dropdown and the target format in the To dropdown, then click Convert or press Ctrl+Enter.

Keyboard shortcuts

These shortcuts are active while focus is inside the Be Water tool window, including inside the input and output editors. Other actions are available from the toolbar buttons.

  • Ctrl+Enter converts input to the selected output format
  • Ctrl+F opens a find bar that searches the focused editor (Enter = next match, Shift+Enter = previous, Esc = close)

The main operations (Convert, Format Input, Copy Output, Open File, Save Output) are also registered as IDE actions — search for "Be Water" in Find Action or assign your own shortcuts in Settings → Keymap.

Toolbar extras

  • Cancel — while a conversion runs, the Convert button becomes Cancel.
  • History — the clock button lists the last 20 conversions; selecting one restores both editors. Restoring saves the current state first, so it can be undone.
  • Soft-wrap — toggles line wrapping in both editors.
  • Split toggle — switches between side-by-side and stacked editors.

Conversion-specific options

Some formats show an options bar below the toolbar. All options are remembered across IDE restarts:

  • CSV output choose between FLAT_FIRST (expands the first array only) and CROSS_JOIN (full Cartesian product). In CROSS_JOIN mode, adjust the Row warning threshold to control when the confirmation dialog appears.
  • CSV / XML input toggle Infer types to convert values that look like numbers, booleans, or null into typed JSON values (leading-zero values like 007 always stay strings).
  • Java POJO toggle Lombok annotations to add @Data, @NoArgsConstructor, and @AllArgsConstructor, and Detect dates to type ISO-8601 values as LocalDate / LocalDateTime / OffsetDateTime.

Input leniency

JSON input tolerates comments, trailing commas, single quotes, and unquoted field names, and truncated JSON is auto-repaired (unclosed brackets and strings are closed). Multi-document YAML ("---"-separated, e.g. Kubernetes manifests) converts to a JSON array.

Swap behavior

The swap button exchanges input and output when the output format can also be parsed as an input format. Generated Java POJO output is output-only, so it cannot be swapped back into the input editor as a source format.
